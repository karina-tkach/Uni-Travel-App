# Сервіс подорожей для студентів

## Загальна інформація

**Сервіс подорожей для студентів** — це вебзастосунок, призначений для комплексного планування подорожей з урахуванням потреб студентської аудиторії. Основна мета проєкту полягає у спрощенні процесу організації подорожей шляхом об’єднання різних туристичних сервісів в єдину платформу.

Система надає можливість шукати та переглядати пропозиції житла, транспорту, закладів харчування й розваг, а також формувати групові маршрути. Особливу увагу приділено підтримці групових подорожей, що дозволяє декільком користувачам спільно планувати поїздку та контролювати її цілісність.

Проєкт передбачає інтеграцію з зовнішніми сервісами бізнес-партнерів для отримання актуальних пропозицій, а також реалізацію системи реєстрації та автентифікації користувачів.

Архітектура застосунку спроєктована з урахуванням вимог масштабованості, безпеки та можливості подальшого розширення функціоналу.

---

## Мета проєкту

- створення єдиної платформи для планування студентських подорожей;
- забезпечення підтримки бюджетних та групових поїздок;
- інтеграція туристичних сервісів і бізнес-партнерів;
- спрощення процесу організації подорожей для студентів.

---

## Основні можливості

- пошук і перегляд житла та транспортних маршрутів;
- формування групових подорожей;
- каталог закладів харчування та розваг;
- реєстрація та автентифікація користувачів;
- особистий кабінет користувача;
- адміністрування та модерація контенту;
- інтеграція з сервісами бізнес-партнерів.

---

## Архітектура

Система побудована з використанням багаторівневої архітектури (**Layered Architecture**) з поділом на такі рівні:

- Presentation Layer — клієнтська частина (React);
- Controller Layer — обробка REST-запитів;
- Service Layer — бізнес-логіка;
- Data Access Layer — взаємодія з базою даних.

Такий підхід забезпечує масштабованість, тестованість і гнучкість інтеграції з зовнішніми сервісами.

---

## Технології

- **Backend:** Java, Spring Boot
- **Frontend:** React
- **Database:** PostgreSQL
- **Authentication:** Google OAuth2, Spring Security
- **API:** REST
- **ORM:** Spring Data JPA

## Запуск застосунку (покроково)

### Передумови

Перед початком переконайтеся, що на вашому комп’ютері встановлено:
- **Java 17+**
- **Node.js 18+**
- **PostgreSQL**
- **npm**

---

### Крок 1. Налаштування змінних середовища

Перед запуском необхідно додати всі конфігураційні параметри.

#### Backend

У backend-модулі створіть або відредагуйте файл `env.properties` та вкажіть необхідні параметри (підключення до БД, OAuth тощо):

```properties
RAPIDAPI_KEY=your-key
OUTSCRAPER_KEY=your-key

DB_HOST=your-host
DB_PORT=your-port
DB_NAME=your-db-name
DB_USER=your-username
DB_PASSWORD=your-password

GOOGLE_CLIENT_ID=your-key
GOOGLE_CLIENT_SECRET=your-key
```
---
### Крок 2. Запуск backend (Spring Boot)
Перейдіть у папку backend-проєкту та запустіть Spring Boot застосунок:

```bash
  ./gradlew bootRun
```
Після запуску backend буде доступний за адресою:
```
http://localhost:8080
```
---
### Крок 3. Встановлення залежностей frontend
Після успішного запуску backend перейдіть у папку клієнтського застосунку:

```bash
  cd app
```
Встановіть усі необхідні бібліотеки:

```bash
  npm install
```
---
### Крок 4. Запуск frontend у режимі розробки
Запустіть клієнтський застосунок у режимі розробки:

```bash
  npm run dev
```
Frontend буде доступний за адресою:
```
http://localhost:5173
```
